# STM32F103C8 + DHT11 + OLED + 蜂鸣器/LED 报警 硬件连接说明

## 硬件清单

| 序号 | 元器件 | 数量 | 说明 |
|------|--------|------|------|
| 1 | STM32F103C8T6 最小系统板 (Blue Pill) | 1 | 主控 MCU |
| 2 | DHT11 温湿度传感器模块 | 1 | 3 针脚：VCC、DATA、GND |
| 3 | 0.96 寸 OLED 显示屏 (SSD1306, I2C) | 1 | 4 针脚：VCC、GND、SCL、SDA |
| 4 | 有源蜂鸣器模块 (低电平触发) | 1 | 3 针脚：VCC、GND、I/O |
| 5 | 5mm LED + 220Ω 限流电阻 | 1 | 或直接使用 LED 模块 |
| 6 | 面包板 + 杜邦线若干 | 若干 | 连接线路 |
| 7 | USB-TTL 下载器 (如 ST-Link) | 1 | 烧录程序 |

## 引脚连接图

### OLED (I2C 接口)

| OLED 引脚 | STM32 引脚 | 说明 |
|-----------|-----------|------|
| VCC | 3.3V | 电源正极 (不能接 5V) |
| GND | GND | 电源地 |
| SCL | **PB6** | I2C 时钟线 (软件模拟) |
| SDA | **PB7** | I2C 数据线 (软件模拟) |

> 注意：OLED 模块通常工作电压为 3.3V，请勿接 5V，以免损坏屏幕。

### DHT11 温湿度传感器

| DHT11 引脚 | STM32 引脚 | 说明 |
|-----------|-----------|------|
| VCC | **3.3V 或 5V** | 电源正极 (DHT11 工作电压 3~5.5V) |
| DATA | **PB0** | 单总线数据线 |
| GND | GND | 电源地 |

> 注意：
> - DATA 线与 VCC 之间建议外接一个 **4.7kΩ ~ 10kΩ** 的上拉电阻（部分模块已自带）。
> - 连接线建议控制在 **20cm 以内**，过长会影响通信稳定性。

### 有源蜂鸣器 — 温度报警 (PB1)

| 蜂鸣器引脚 | STM32 引脚 | 说明 |
|-----------|-----------|------|
| VCC | **3.3V 或 5V** | 电源正极 (有源蜂鸣器大多支持宽电压) |
| I/O | **PB1** | 控制信号 (低电平响，高电平静) |
| GND | GND | 电源地 |

> **行为**：温度超过阈值（默认 35°C）时，蜂鸣器以 **300ms 响 + 700ms 停** 的节奏循环报警。

### LED 报警指示灯 — 湿度报警 (PB2)

| LED 引脚 | STM32 引脚 | 说明 |
|-----------|-----------|------|
| 阳极 (+ 长脚) | **PB2** | GPIO 控制信号 |
| 阴极 (- 短脚) | **GND** (经 220Ω 电阻) | 限流电阻串联 |

连接方式：

`
PB2 ──── 220Ω ──── LED(+) ──── LED(-) ──── GND
`

> **行为**：湿度超过阈值（默认 85%RH）时，LED **保持常亮**，不闪烁。

## 引脚汇总

| 功能 | STM32 引脚 | 外设引脚 | 报警条件 |
|------|-----------|---------|---------|
| OLED SCL | **PB6** | OLED SCL | — |
| OLED SDA | **PB7** | OLED SDA | — |
| DHT11 DATA | **PB0** | DHT11 DATA | — |
| 蜂鸣器 I/O | **PB1** | BUZZER I/O | 温度 > 35°C → 300ms响+700ms停 |
| LED 报警 | **PB2** | LED → 220Ω → GND | 湿度 > 85%RH → 常亮 |
| 电源 3.3V | **3.3V** | OLED VCC | — |
| 电源 3.3V/5V | **3.3V/5V** | DHT11 VCC, 蜂鸣器 VCC | — |
| GND | **GND** | 所有模块共地 | — |

## 报警阈值说明

代码中预设的报警阈值在 Hardware/Alert/Alert.h 中定义：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| TEMP_ALARM_THRESHOLD | **35** (°C) | 温度超过此值 → 蜂鸣器响 |
| HUMI_ALARM_THRESHOLD | **85** (%RH) | 湿度超过此值 → LED 常亮 |

用户可以按需修改这两个宏定义的数值。温度报警和湿度报警完全独立，互不影响。

## 烧录与下载

1. 使用 **ST-Link** 或 **USB-TTL (串口)** 连接 STM32F103C8 的相应引脚
2. 在 Keil uVision 中打开本项目（project.uvprojx）
3. 编译后下载到开发板

## 注意事项

1. **共地**：所有模块的 GND 必须连接到 STM32 的 GND，否则无法通信。
2. **上电顺序**：建议先给 STM32 上电，待系统初始化完成后再给传感器供电。
3. **DHT11 采样间隔**：DHT11 数据手册建议两次读取间隔不小于 1 秒，代码中已设置为 2 秒。
4. **DHT11 连线距离**：DHT11 与 MCU 之间的连线不宜过长，建议不超过 20cm。
5. **OLED 电压**：OLED 模块工作电压为 3.3V，切勿接 5V 引脚。
6. **蜂鸣器类型**：本项目代码默认使用 **低电平触发** 有源蜂鸣器。若使用高电平触发型，需将 Alert.c 中 Alert_Buzzer_ON() 和 Alert_Buzzer_OFF() 的电平逻辑取反。
